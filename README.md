# Kubernetes
🎯 Learning Progression

This project reflects a deliberate transition across three stages of learning:

1. Conceptual Familiarity

Initial exposure to Kubernetes concepts such as Pods, Deployments, and Services established a high-level understanding of container orchestration and its role in distributed systems.

2. Hands-On Implementation

Through guided tutorials and direct interaction with tools like kubectl, Minikube, and kind, this project moves beyond theory into practical application.
This includes:

* Running and inspecting workloads in a cluster
* Deploying containerized Spring services
* Exploring the Kubernetes Dashboard and CLI workflows

At this stage, the focus is on developing intuition—understanding how Kubernetes behaves in response to commands and configurations.

3. Deepening Operational Understanding (In Progress)

The next phase focuses on building a more rigorous understanding of Kubernetes by:

* Studying official documentation in depth
* Analyzing the purpose and effect of each command and configuration
* Observing system behavior through debugging (kubectl describe, logs, events)
* Strengthening knowledge of how components interact internally

AI tools are used to:

* Clarify concepts quickly
* Fill knowledge gaps
* Accelerate iteration and experimentation

The objective is to move toward confident, independent usage of Kubernetes fundamentals, with a clear understanding of both the mechanics and intent behind each operation.

⸻

This progression represents a shift from:

“Recognizing Kubernetes concepts”

to:

“Effectively using and reasoning about Kubernetes in practice”
